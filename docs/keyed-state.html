linkKey Tracking

When you have an array of objects, it often happens that you are interested on a particular entity based on some identifier, regardless of its position within the array:

1import { State } from 'rxdeep';

2

3const state = new State([

4  { id: 101, name: 'Jack', age: 32 },

5  { id: 102, name: 'Jill', age: 67 },

6  ...

7]);


In this example, you might want a reactive state object that reflects state of Jack, independent of where Jack sits in the array. The KeyedState class allows you to track entities of an array using some key function which helps identify each entitiy:

1import { State, KeyedState } from 'rxdeep';

2

3const state = new State([

4  { id: 101, name: 'Jack', age: 32 },

5  { id: 102, name: 'Jill', age: 67 },

6  ...

7]);

8

9const keyed = new KeyedState(state, person => person.id);

10keyed.key(101).subscribe(console.log);


In this example, the key function is person => person.id, i.e. each person is identified by their .id property. .key() method behaves like State.sub() method, except that it represents state of the entity matching given key instead of the entity on given index / property:

1state.value = [{ id: 103, name: 'James', age: 21 },  // --> no changes

2               ...state.value];                      // --> no changes

3keyed.value = [{ id: 103, name: 'James', age: 21},   // --> no changes

4               { id: 101, name: 'Jack', age: 32 }];  // --> no changes

5state.value = [{ id: 101, name: 'Jack', age: 33 }];  // --> change!!

6

7// Logs only for the initial value and the last change:

8// > { id: 101, name: 'Jack', age: 32 }

9// > { id: 101, name: 'Jack', age: 33 }


The .key() method returns a typical State, so you can similarly use them to set values:

1keyed.key(101).sub('age').value = 34;

2

3// Logs:

4// > { id: 101, name: 'Jack', age: 34 }

touch_app NOTICE

Similar to a sub-state, it is highly recommended to keep a keyed substate (result of .key() method) subscribed (having called its .subscribe() method) before issueing changes to it, to ensure that the state is in sync with the change history.

linkIndex Tracking

You can also track the index of a particular key within the array using .index() method:

1import { State, KeyedState } from '../src';

2

3const state = new State([

4  { id: 101, name: 'Jack', age: 32 },

5  { id: 102, name: 'Jill', age: 67 },

6]);

7const keyed = new KeyedState(state, person => person.id);

8keyed.index(101).subscribe(console.log);

9

10state.value = [{ id: 103, name: 'James', age: 21 },

11               ...state.value];

12keyed.value = [{ id: 103, name: 'James', age: 21},

13               { id: 101, name: 'Jack', age: 32 }];

14state.value = [{ id: 101, name: 'Jack', age: 33 }];

15

16// Logs:

17// > 0

18// > 1

19// > 0

linkDetailed Array Changes

The KeyedState class also comes with .changes() method which provides a detailed change profile of the array in terms of additions, deletions and moved items:

1import { State, KeyedState } from '../src';

2

3const state = new State([

4  { id: 101, name: 'Jack', age: 32 },

5  { id: 102, name: 'Jill', age: 67 },

6]);

7const keyed = new KeyedState(state, person => person.id);

8keyed.changes().subscribe(console.log);

9

10state.value = [{ id: 103, name: 'James', age: 21 },

11               ...state.value];

12// Logs:

13// > {

14// >   additions: [{index: 0, item: {id: 103, name: 'James', age: 21}}],

15// >   deletions: [],

16// >   moves: [

17// >     {oldIndex: 0, newIndex: 1, item: {id: 101, name: 'Jack', age: 32}},

18// >     {oldIndex: 1, newIndex: 2, item: {id: 102, name: 'Jill', age: 67}}

19// >   ]

20// > }

21

22keyed.value = [{ id: 103, name: 'James', age: 21},   // --> no changes

23               { id: 101, name: 'Jack', age: 32 }];  // --> no changes

24// Logs:

25// > {

26// >   additions: [],

27// >   deletions: [{index: 2, item: {id: 102, name: 'Jill', age: 67}}],

28// >   moves: []

29// > }

30

31state.value = [{ id: 101, name: 'Jack', age: 33 }];  // --> change!!

32// Logs:

33// > {

34// >   additions: [],

35// >   deletions: [{index: 0, item: {id: 103, name: 'James', age: 21}}],

36// >   moves: [{oldIndex: 1, newIndex: 0, item: {id: 101, name: 'Jack', age: 32}}]

37// > }

38

39keyed.key(101).sub('age').value = 34;

40// Logs:

41// > {

42// >   additions: [],

43// >   deletions: [],

44// >   moves: []

45// > }
